# Implementation Plan - CosmOS: The Divine Repair Tool

A morbidly beautiful "Sacred OS" idle game where you succeed a retired deity to repair the universe.

## 1. Aesthetic & UI (Retro-Divine Hybrid)
- **Visual Style**: Windows 98 chrome with Windows 7 transparency effects (Aero-Divine).
- **Color Palette**: Celestial Blues, Gold accents, "Paper" White, and Deep Void Black.
- **Typography**: `Fixedsys`, `MS Sans Serif`, or `Times New Roman` for that 90s feel.
- **Components**:
    - Window Shell: Classic title bars, [X][_][?] buttons.
    - Taskbar: "Start" (or "Genesis") menu, System Clock (Universal Epoch).
    - Desktop Icons: Recycle Bin (The Void), My Computer (This Universe).

## 2. Core Mechanics (The Idler)
- **Primary Resource**: `Praise` (Generated via prayers/sentient life).
- **Secondary Resource**: `Offerings` (Generated via miracles).
- **The Main App**: "Universal Engine Console" - where 99% of the gameplay happens.
- **Skill Tree**: "Divine Mandates" - unlocking fundamental laws of physics/magic.
- **Regions**: Dimensions like "The Primordial Soup", "The Solar Foundry", "The Astral Library".

## 3. Technical Architecture
- **State Management**: Centralized store with localStorage persistence.
- **Window Management**: A lightweight manager for z-index, dragging, and focus.
- **Logic Loop**: `requestAnimationFrame` for game ticks.

## 4. Phase 1: The Boot Sequence & Desktop
- Initialize the HTML shell.
- Create the CSS Design System (The "Sacred Shell").
- Implement the Window Manager.
- Create the "Start/Genesis" menu.

## 5. Phase 2: The Divine Console (Core Loop)
- Basic resource generation.
- The pulsing "Feynman" style visualization for resource flow.
- Upgrade buttons in a classic Win98 list format.

## 6. Phase 3: Dimensional Expansion
- Adding the "Dimension Explorer" app.
- Specializing resources per region.
